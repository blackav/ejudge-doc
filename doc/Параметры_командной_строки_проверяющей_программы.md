Число параметров командной строки зависит от настроек задачи.
Проверяющей программе может передаваться от двух до шести параметров
командной строки.

- Путь к файлу с входными данными теста. Путь указывает на файл с
  входными данными, находящийся в каталоге для тестов задачи в системе.
  Поэтому случайные или намеренные изменения, которые тестируемая
  программа сделала в своей копии входного файла, будут проигнорированы.
- Путь к файлу, в котором находится результат работы тестируемой
  программы.
- Путь к файлу, в котором находится эталонный ответ для данного теста
  (только в случае, если для данной задачи установлена конфигурационная
  переменная [`use_corr`](Serve.cfg:problem:use_corr "wikilink")).
- Путь к [файлу дополнительной информации о
  тесте](.inf-файлы "wikilink") для данного теста (только в случае, если
  для данной задачи установлена конфигурационная переменная
  [`use_info`](Serve.cfg:problem:use_info "wikilink")).
- Путь к рабочему каталогу тестируемой программы (только в случае, если
  для данной задачи установлена конфигурационная переменная
  [`use_tgz`](Serve.cfg:problem:use_tgz "wikilink")).
- Путь к эталонному содержимому рабочего каталога (только в случае, если
  для данной задачи установлена конфигурационная переменная
  [`use_tgz`](Serve.cfg:problem:use_tgz "wikilink")).

Таким образом, в наиболее типичной конфигурации проверяющая программа
получает три аргумента командной строки:

- argv\[1\] - имя файла с входными данными
- argv\[2\] - имя файла с результатом работы программы
- argv\[3\] - имя файла с эталонным ответом
