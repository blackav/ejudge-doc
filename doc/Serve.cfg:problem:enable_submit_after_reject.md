Если конфигурационная переменная установлена в ненулевое значение,
участник может сдавать решения задачи, по которой у него есть посылки в
статусе Rejected и нет посылок в статусе OK, независимо от даты закрытия
задачи (параметр [`deadline`](serve.cfg:problem:deadline "wikilink")).

Например, на момент закрытия задачи для сдачи у участника посылка
находилась в статусе Pending Review. Уже после закрытия задачи посылка
была просмотрена и получила статус Rejected. Тогда, если переменная
`enable_submit_after_reject` установлена, участник получает возможность
сдавать эту задачу несмотря на истечение срока сдачи. Как только
участник получает OK по данной задаче, возможность сдачи закрывается.

Значение наследуется из абстрактной задачи.

Поддерживается начиная с версии
[3.6.0](изменения_в_версии_3.6.0 "wikilink")

С версии [3.6.1](изменения_в_версии_3.6.1 "wikilink") эта переменная
влияет на вычисление штрафа, заданного с помощью
[`date_penalty`](serve.cfg:problem:date_penalty "wikilink"). Если эта
конфигурационная переменная установлена, то для вычисления штрафа за
посылку используется <i>эффективное время</i> сдачи данной задачи данным
пользователем, а не время данной посылки. Эффективное время - это
минимальное время среди всех посылок данного пользователя по данной
задаче со статусом [Rejected](RJ "wikilink").
