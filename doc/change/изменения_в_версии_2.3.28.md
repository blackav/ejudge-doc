Время выхода: 31 октября 2013 г.

Исправление ошибок, некоторые из которых приводили к падению системы
ejudge.

### [ej-compile](../tool/ej-compile.md)

Добавлены языки программирования gcc-32 и g++-32. Это - 32-битные версии
компиляторов gcc и g++. Кроме того, программы линкуются динамически, а
не статически, что значительно уменьшает размер исполняемого файла и,
соответственно, время пересылки файлов по сети.

Добавлены языки программирования fpc-32, fbc-32, clang-32, clang++-32.
Это — 32-битные версии соответствующих компиляторов.

Для компилятора gccgo обязательно добавляется ключ -g (отладочная
информация), так как в противном случае скомпилированные программы не
запускаются.

### [ej-contests](../tool/ej-contests.md)

Существенно ускорена работа при больших размерах таблицы посылок.

Исправлена ошибка в автоопределении кодировки UTF-16. В некоторых
случаях ошибочно определялся неверный порядок байт.

Улучшен контроль прав доступа при просмотре и отправлении сообщений.

Добавлена глобальная конфигурационная переменная
[`time_limit_retry_count`](../serve.cfg/global/time_limit_retry_count.md).

Добавлена глобальная конфигурационная переменная
[`score_n_best_problems`](../serve.cfg/global/score_n_best_problems.md).

Добавлена глобальная конфигурационная переменная
[`checker_locale`](../serve.cfg/global/checker_locale.md).

Реализована поддержка конфигурационной переменной
[`max_user_run_count`](../serve.cfg/problem/max_user_run_count.md)
раздела описания задачи.

Добавлена конфигурационная переменная
[`show_checker_comment`](../serve.cfg/problem/show_checker_comment.md)
раздела описания задачи.

Добавлена конфигурационная переменная
[`lang_max_vm_size`](../serve.cfg/problem/lang_max_vm_size.md)
раздела описания задачи.

Добавлена конфигурационная переменная
[`lang_max_stack_size`](../serve.cfg/problem/lang_max_stack_size.md)
раздела описания задачи.

Добавлена конфигурационная переменная
[`super_run_dir`](../serve.cfg/language/super_run_dir.md) раздела
описания языка программирования.

### [ej-super-run](../tool/ej-super-run.md)

Добавлена опция командной строки -m для сохранения локальных копий
файлов с тестами в случае работы по сети.

Если включена поддержка определения ошибки [Memory Limit
Exceeded](../verdict/memory_Limit_Exceeded.md) (см. глобальную
конфигурационную переменную
[`enable_memory_limit_error`](../serve.cfg/global/enable_memory_limit_error.md)),
сообщения, выдаваемые java-программой на стандартный поток ошибок
анализируются на наличие вывода сообщения об ошибке
java.lang.OutOfMemoryError. В этом случае диагностируется ошибка memory
limit exceeded.

### [ej-super-server](../tool/ej-super-server.md)

Поддерживается импорт турниров из системы Polygon.

### [ej-polygon](../tool/ej-polygon.md)

Поддерживается текущий протокол обмена с системой Polygon.

Поддерживается загрузка задач по идентификатору турнира.

Поддерживаются интерактивные задачи.

Задача не скачивается, если есть несохраненные изменения (uncommitted
changes).

### [gvaluer](../gvaluer.md)

Поддерживаются элементы конфигурационного файла sets_marked_if_passed,
pass_if_count.

### [Стандартные проверяющие программы](../стандартные_проверяющие_программы.md)

Поддерживается переменная окружения `EJUDGE_LOCALE`, которая позволяет
задавать язык вывода сообщений в проверяющих программах. Например, если
эта переменная установлена в значение `ru` (или `russian`), сообщения
будут выводиться на русском языке.

### [libchecker](../libchecker.md)

Добавлена функция `checker_l10n_prepare`, которая включает локализацию
выводимых сообщений. Язык локализации берется из переменной окружения
EJUDGE_LOCALE.
