| **Имя переменной**:        | **`max_line_length`**                   |     |
| **Содержится в:**          | [`global`](../global.md) |     |
| **Используется:**          | `run`                                   |     |
| **Тип содержимого:**       | *integer*                               |     |
| **Может отсутствовать:**   | *да*                                    |     |
| **Значение по умолчанию:** | `4096`                                  |     |
| **Может повторяться:**     | *нет*                                   |     |

**Редактирование элемента:** страница *"Editing contest"*, вкладка
*"Global settings (serve.cfg)"*, блок *"Advanced settings"*, поле
*"Maximal line length to be included into testing protocol"*.

**Описание.** Данная конфигурационная переменная задаёт ограничение на
максимальную длину строки в файле протокола тестирования программы. Если
длина какой-либо строки в файле, который копируется в файл протокола
тестирования, превышает это значение, такая строка заменяется на строку
`<string is too long>`. Ограничение максимальной длины строки действует
для файлов с входными данными тестов, файлов с правильными ответами к
тестам, файлов с выводом тестируемой программы, файлов с выводом в поток
ошибок тестируемой программы и программы, проверяющей правильность
решения.

**Пример.**

`max_line_length = 1024`
