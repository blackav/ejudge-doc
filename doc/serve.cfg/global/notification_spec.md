# `notification_spec`

Навигация: [Система ejudge](../../система_ejudge.md)/[Использование](../../использование.md)/[Конфигурационные файлы](../../конфигурационные_файлы.md)/[serve.cfg](../../serve.cfg.md)/[Глобальные конфигурационные параметры](../global.md)/[`notification_spec`](notification_spec.md)

Поддерживается начиная с версии
[3.13.0](../../change/изменения_в_версии_3.13.0.md).

В этом конфигурационном параметре можно указать очередь сообщений, куда ejudge будет посылать информацию об изменении статуса посылок (подробнее см. [здесь](../../нотификации_во_внешние_системы.md)). Формат переменной:

```
notification_spec = "<notify_driver>:<notify_kind>:<notify_queue>"
```

Данные параметры описаны [здесь](../../нотификации_во_внешние_системы.md#L24).
