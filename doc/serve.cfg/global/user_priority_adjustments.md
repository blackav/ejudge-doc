**Описание.** Данная конфигурационная переменная позволяет задать
изменение приоритета тестирования (см. описание глобальной
конфигурационной переменной
[`priority_adjustment`](priority_adjustment.md))
для некоторого пользователя. Определение данной переменной может быть
повторено несколько раз, задавая таким образом изменения приоритета для
нескольких пользователей.

Спецификация приоритета имеет следующий вид:

`LOGIN PRIO-CHANGE`

Здесь *LOGIN* — это регистрационное имя пользователя, а *PRIO-CHANGE* —
целое число, изменение приоритета. Отрицательное изменение приоритета
означает повышение приоритета относительно базового значения, а
положительное — понижение приоритета.

**Пример.**

`user_priority_adjustment = "admin -10"`  
`user_priority_adjustment = "vasya 5"`
