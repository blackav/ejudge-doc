# `enable_remote_cache`

Навигация: [Система ejudge](../../система_ejudge.md)/[Использование](../../использование.md)/[Конфигурационные файлы](../../конфигурационные_файлы.md)/[serve.cfg](../../serve.cfg.md)/[Глобальные конфигурационные параметры](../global.md)/[`enable_remote_cache`](enable_remote_cache.md)

Типичный путь поступившего на тестирование файла выглядит следующим
образом:

- исходный файл передается через каталоги обмена от
  [ej-contests](../../tool/ej-contests.md) к
  [ej-compile](../../tool/ej-compile.md). Если компоненты находятся на
  разных хостах, файл копируется по сети.
- после компиляции скомпилированный файл передается через каталоги
  обмена от [ej-compile](../../tool/ej-compile.md) к
  [ej-contests](../../tool/ej-contests.md). Если компоненты находятся на
  разных хостах, файл копируется по сети.
- скомпилированный файл передается через каталоги обмена от
  [ej-contests](../../tool/ej-contests.md) к
  [ej-super-run](../../tool/ej-super-run.md). Если компоненты находятся на
  разных хостах, файл копируется по сети.

Размер скомпилированного файла может быть весьма большим (несколько
мегабайт), например, для программ на kotlin или go, и копирование
исполняемого файла по сети туда и обратно может потребовать заметного
времени, сравнимого со временем тестирования.

В случае, когда и компонент компиляции
[ej-compile](../../tool/ej-compile.md), и компонент тестирования
[ej-super-run](../../tool/ej-super-run.md) выполняются на одном и том же
хосте, отличном от хоста [ej-contests](../../tool/ej-contests.md), и только
на этом хосте, передавать результат компиляции по сети на сервер
[ej-contests](../../tool/ej-contests.md) и обратно не имеет смысла.

В этом случае можно использовать локальное кеширование результатов
компиляции. Компонент [ej-compile](../../tool/ej-compile.md) не скопирует
файл на сервер, а положит его в локальный каталог, из которого он будет
забран компонентом [ej-super-run](../../tool/ej-super-run.md).

Для включения локального кеширования глобальная конфигурационная
переменная `enable_remote_cache` должна быть установлена в положительное
значение. Кроме того, при запуске компонентов компиляции и тестирования
ejudge должна быть указана опция `--local-cache`.

Пример:

```text
enable_remote_cache
```

Смотри также раздел [параллельное тестирование](../../параллельное_тестирование.md).

Поддерживается начиная с версии
[3.11.0](../../change/изменения_в_версии_3.11.0.md).
