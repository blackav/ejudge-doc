|                            |                                         |     |
|----------------------------|-----------------------------------------|-----|
| **Имя переменной**:        | **`run_log_file`**                      |     |
| **Содержится в:**          | [`global`](../global.md) |     |
| **Используется:**          | `serve`                                 |     |
| **Тип содержимого:**       | *путь к файлу*                          |     |
| **Может отсутствовать:**   | *да*                                    |     |
| **Значение по умолчанию:** | `run.log`                               |     |
| **Может повторяться:**     | *нет*                                   |     |

**Описание.** Данная конфигурационная переменная устанавливает путь к
файлу, в котором программа serve хранит базу посылок пользователей.
Полный путь к файлу определяется по следующим правилам.

- Если значение переменной `run_log_file` не задано, используется
  значение `run.log`.

<!-- -->

- Если значение переменной `run_log_file` начинается с символа ’/’, то
  есть значение переменной задаёт полный абсолютный путь к файлу, этот
  путь используется без изменений.

<!-- -->

- Если значение переменной `run_log_file` не начинается с символа ’/’,
  полный путь к файлу образуется конкатенацией значения переменной
  [`var_dir`](var_dir.md) и значения переменной
  `run_log_file`.

**Внимание.** Изменять значение данной конфигурационной переменной не
рекомендуется.
