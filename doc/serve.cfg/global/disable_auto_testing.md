| **Имя переменной**:        | **`disable_auto_testing`**              |     |
| **Содержится в:**          | [`global`](../global.md) |     |
| **Используется:**          | `serve`                                 |     |
| **Тип содержимого:**       | *boolean*                               |     |
| **Может отсутствовать:**   | *да*                                    |     |
| **Значение по умолчанию:** | *false*                                 |     |
| **Может повторяться:**     | *нет*                                   |     |

**Редактирование элемента:** страница *"Editing contest"*, вкладка
*"Global settings (serve.cfg)"*, блок *"Advanced settings"*, поле
*"Disable automatic testing of submissions"*.

**Описание.** Если данная конфигурационная установлена в *true*,
автоматическое тестирование решений задач, посылаемых участниками,
отключено. В этом случае решение участника получает статус “Accepted for
testing” и не отправляется на компиляцию и тестирование. Администратор
турнира должен явно запустить тестирование решения участника, выбрав
пункт “Rejudge” для данного решения. Данная конфигурационная переменная
устанавливает значение, используемое в случае, если описание задачи не
определяет конфигурационную переменную
[`disable_auto_testing`](disable_auto_testing.md)
самостоятельно.

**Пример.**

`disable_auto_testing`
