|----------------------------|-------------------------------------------|
| **Имя переменной**:        | **`priority_adjustment`**                 |
| **Содержится в:**          | [`problem`](serve.cfg:problem.md) |
| **Используется:**          | `serve`                                   |
| **Тип содержимого:**       | *integer*                                 |
| **Может отсутствовать:**   | *да*                                      |
| **Наследуется:**           | *да*                                      |
| **Значение по умолчанию:** | `0`                                       |
| **Может повторяться:**     | *нет*                                     |
| **Версия `ejudge`:**       | начиная с 2.1.29.                         |

**Описание.** Данная конфигурационная переменная позволяет задать
поправку к приоритету тестирования для данной задачи. Значение поправки
будет прибавлено к базовому значению приоритета тестирования (см.
описание глобальной конфигурационной переменной
[`priority_adjustment`](serve.cfg:global:priority_adjustment.md))
при вычислении приоритета тестирования посылки. Положительное значение
поправки уменьшает приоритет, а отрицательное — увеличивает.

**Пример.**

`priority_adjustment = 2`
