Данный параметр позволяет задать минимальный интервал времени, который
должен пройти между двумя [запусками на пользовательских
данных](Запуск_на_пользовательских_данных.md) одного
пользователя в одном турнире.

Положительное значение параметра задает минимальное время, если параметр
равен 0, то проверка ограничения по времени отключается. Значение этого
параметра по умолчанию равно 5.

<b>Пример:

`time_between_submits = 10`

Поддерживается начиная с версии
[3.10.0](изменения_в_версии_3.10.0.md).
