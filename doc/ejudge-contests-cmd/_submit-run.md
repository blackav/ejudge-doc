[Команды](_COMMAND.md)

Команда submit-run отправляет решение на проверку. Если сессия была
создана с правами обычного пользователя, то выполняются все стандартные
проверки на допустимости отправки посылки.

Использование:

`ejudge-contests-cmd CONTEST-ID submit-run [OPTIONS] SESSION-FILE PROB-SHORT-NAME LANG-SHORT-NAME SOURCE-FILE [VARIANT]`

Поддерживаются все [стандартные опции
команд](_COMMAND-OPTIONS.md). В частности,
если указана опция `--session`, то аргумент `SESSION-FILE` — это не
файл, в котором записан сессионный ключ, а сам сессионный ключ.

Здесь `PROB-SHORT-NAME` — это краткое имя задачи. `LANG-SHORT-NAME` —
краткое имя языка программирования. Если указание языка программирования
для этой задачи не требуется (т. е., например, output-only задача), то
значение этого параметра произвольно, например `N/A`. `SOURCE-FILE` —
это имя сдаваемого файла. Если в качестве имени файла указать `STDIN`,
сдаваемый файл будет считан со стандартного потока ввода. `VARIANT` —
это вариант задачи. Указание варианта допускается только для
привилегированных сдач, и если данная задача поддерживает варианты.
