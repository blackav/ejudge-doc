|                            |                                         |
|----------------------------|-----------------------------------------|
| **Имя переменной**:        | **`error_file`**                        |
| **Содержится в:**          | [`tester`](serve.cfg:tester "wikilink") |
| **Используется:**          | `run`                                   |
| **Тип содержимого:**       | *имя файла*                             |
| **Может отсутствовать:**   | *да*                                    |
| **Наследуется:**           | *да*                                    |
| **Значение по умолчанию:** | `error`                                 |
| **Может повторяться:**     | *нет*                                   |

**Описание.** Данная конфигурационная переменная устанавливает имя
файла, в который перенаправляется вывод на стандартный поток ошибок
тестируемой программы. После окончания тестирования содержимое этого
файла вставляется в протокол тестирования. Переменная не используется
(то есть перенаправления не происходит), если конфигурационная
переменная [`no_redirect`](serve.cfg:tester:no_redirect "wikilink")
установлена в значение *true*. Файл перенаправления потока ошибок
создаётся в рабочем каталоге тестирования, задаваемом конфигурационной
переменной [`check_dir`](serve.cfg:tester:check_dir "wikilink").
Значение переменной `error_file` не должно содержать символа ’/’, то
есть должно быть именем файла в текущем каталоге.
