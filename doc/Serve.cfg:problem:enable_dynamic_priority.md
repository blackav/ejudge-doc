Если у задачи установлен данный конфигурационный параметр, приоритет
тестирования посылки пользователя зависит от числа других посылок того
же пользователя по той же задаче, которые в текущий момент находятся в
состоянии компиляции или тестирования. Поправка к приоритету
(priority_adjustment) вычисляется по следующей формуле:

`priority_adjustment = (testing_count - 1) / 2 + 3;`  
`if (priority_adjustment > 12) priority_adjustment = 12;`

Здесь `testing_count` — это число посылок, уже находящихся в очереди на
компиляцию и тестирование. Чем больше поправка к приоритету, тем ниже
приоритет тестирования, то есть такая посылка будет находиться в очереди
тестирования дальше от начала.

Параметр наследуется из абстрактной задачи.

Пример:

`[problem]`  
`# ...`  
`enable_dynamic_priority`

Поддерживается начиная с версии
[3.10.2](изменения_в_версии_3.10.2 "wikilink").
