Время выхода: 1 декабря 2014 г.

Ревизия SVN: 8787

Библиотека [libdwarf](http://www.prevanders.net/dwarf.html) включена в
архив исходных кодов ejudge. Теперь при сборке ejudge эта библиотека
собирается автоматически. Системная библиотека никогда не используется.

Мелкие улучшения в пользовательском интерфейсе.

### [ej-contests](ej-contests "wikilink")

Добавлена поддержка [турниров с
токенами](Турниры_с_токенами "wikilink"). Для настройки турниров с
токенами используются новые конфигурационные переменные задач
[`enable_tokens`](serve.cfg:problem:enable_tokens "wikilink"),
[`tokens`](serve.cfg:problem:tokens "wikilink"),
[`token_open_tests`](serve.cfg:problem:token_open_tests "wikilink").

Исправлена ошибка, из-за которой значение глобальной конфигурационной
переменной
[`enable_max_stack_size`](serve.cfg:global:enable_max_stack_size "wikilink")
не передавалось на тестирование.

Добавлена конфигурационная переменная задачи
[`autoassign_variants`](serve.cfg:problem:autoassign_variants "wikilink"),
которая включает режим автоматического назначения вариантов
пользователям.

Добавлена конфигурационная переменная задачи
[`start_cmd`](serve.cfg:problem:start_cmd "wikilink"), которая позволяет
задать вспомогательную программу для запуска решений участников на
тестирование.

Очередь посылок на компиляцию и тестирование упорядочивается по времени
создания запроса, а не по номеру посылки, как в предыдущих версиях.

### [ej-polygon](ej-polygon "wikilink")

Добавлена конфигурационная опция 'fetch_latest_available'. Если данная
опция установлена, то ej-polygon не пытается пересобрать пакет для linux
самой последней версии задачи, а использует уже готовый пакет для linux
от самой последней из предыдущих версий, у которых пакет для linux был
собран.
