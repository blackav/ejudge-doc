Проверяющая программа информирует тестирующую систему ejudge о вердикте
проверки с помощью кода завершения. Допустимые коды завершения
перечилены ниже:

- 0 - OK, правильный ответ
- 1 - [WA](verdict/WA.md) (с версии 2.3.21)
- 2 - [PE](verdict/PE.md) (с версии 2.3.21)
- 4 - [PE](verdict/PE.md), presentation error, [ошибка неправильного
  формата
  результата](verdict/ошибка_неправильного_формата_результата.md)
- 5 - [WA](verdict/WA.md), wrong answer, [неправильный
  ответ](verdict/неправильный_ответ.md)

Любые другие коды завершения программы, равно как и завершение
проверяющей программы из-за сигнала, рассматриваются как ошибка в работе
самой проверяющей программы.

В программах на C или C++ завершить работу программы с указанным кодом
завершения можно либо вызвав стандартную функцию exit, либо вернув код
завершения как возвращаемое значение из функции main с помощью оператора
return.

Программы на Pascal или Delphi завершаются с кодом 0 по умолчанию. Либо
можно вызвать процедуру halt, передав ей нужный код завершения.

Программы, использующие библиотеку [libchecker](libchecker.md),
могут использовать функции
[checker_OK](libchecker/checker_OK.md),
[fatal_PE](libchecker/fatal_PE.md) или
[fatal_WA](libchecker/fatal_WA.md).

Программы, использующие библиотеку [testlib](testlib.md), могут
использовать процедуру quit.

Начиная с версии 2.3.21 в качестве кода возврата проверяющих программ
допускается 1 (WA) и 2 (PE).
