Система ejudge включает в себя несколько программ:
[`register`](register.md)`, `[`users`](users.md)`, `[`master`](master.md)`, `[`judge`](judge.md)`, `[`team`](team.md)`, `[`serve-control`](serve-control.md)
— запускаемых как CGI-программы. Каждая из этих программ позволяет
ограничивать диапазон IP-адресов клиента, которым разрешается
использование CGI-программы.

В текущей версии системы ограничение диапазонов возможно только с
помощью указания IPv4-адресов клиентов. Ни DNS-имена, ни IP протокол
версии 6 в текущей версии не поддерживаются.

Кроме того, каждый турнир может накладывать дополнительные ограничения
на допустимые IP-адреса для всех CGI-программ. Эти ограничения
указываются в конфигурационном файле турнира
[`contest.xml`](contest.xml.md).

Спецификация ограничения адреса может либо задавать конкретный IP-адрес,
либо задавать адрес сети классов A, B или C. Задание подсетей в сетях
этих классов в настоящее время не поддерживается. Допустимые
спецификации ограничения IP-адреса перечислены в таблице [Допустимые
спецификации ограничения
IP-адреса](Допустимые_спецификации_ограничения_IP-адреса.md).

Синтаксис задания IP-ограничений отличается для CGI-программ,
использующих конфигурационные файлы в формате XML
([`register`](register.md)`, `[`users`](users.md)`, `[`serve-control`](serve-control.md)),
и для CGI-программ, использующих традиционные текстовые файлы
([`master`](master.md)`, `[`judge`](judge.md)`, `[`team`](team.md)).
Далее рассматриваются оба варианта. В конфигурационном файле турнира
[`contest.xml`](contest.xml.md) ограничения задаются в формате
XML.
